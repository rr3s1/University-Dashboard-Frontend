## <a name="introduction">🤖 Introduction</a>

A high-performance academic management platform built on the PERN stack (PostgreSQL, Express, React, Node.js). This multi-role system—designed for Admins, Teachers, and Students—uses a decoupled architecture to deliver a modular React frontend via Refine and a powerful Node.js backend. With Drizzle ORM and Neon PostgreSQL, the system ensures type-safe data handling for class orchestration, automated scheduling, and real-time institutional analytics.

## <a name="tech-stack">⚙️ Tech Stack</a>

- **Frontend**: React, Refine, Tailwind CSS, shadcn/ui, TypeScript, Zod
- **Backend**: Node.js, Express.js, Arcjet, Better Auth, Cloudinary
- **Database**: PostgreSQL (Neon), Drizzle ORM
- **Dev Tools**: CodeRabbit, Site24x7

## <a name="features">🔋 Features</a>

👉 **Role-Based Access Control**: A secure authentication layer using **Better Auth** and **Arcjet** that provides distinct, protected dashboards for Students, Teachers, and Admins.

👉 **Real-Time Institutional Analytics**: A centralized dashboard offering live insights into student enrollment, faculty metrics, and active course statistics.

👉 **Curriculum Management**: Effortless control over academic subjects, allowing for rapid filtering and detailed workload tracking for educators.

👉 **Departmental Governance**: Organizes the academic structure by grouping subjects and faculty into specific departments for streamlined management.

👉 **Advanced Faculty Directory**: A searchable, paginated directory of staff featuring profile management via **Cloudinary** and full schedule visibility.

👉 **Intelligent Class Orchestration**: Powered by **Drizzle ORM**, this engine allows Admins to manage complex session scheduling, seat capacities, and teacher assignments.

👉 **Secure Enrollment Workflow**: A streamlined system where students join specific courses instantly using a unique 6-8 digit access code.

and many more, including high-level code architecture and component reusability.

## <a name="quick-start">🤸 Quick Start</a>

Follow these steps to set up the project locally.

**Prerequisites**

Ensure you have the following installed:
- [Git](https://git-scm.com/)
- [Node.js](https://nodejs.org/en)
- [npm](https://www.npmjs.com/)

**Cloning the Repository**

```bash
git clone https://github.com/rr3s1/University-Dashboard-Frontend.git
cd University-Dashboard-Frontend
```

**Installation**

Install the project dependencies using npm:

```bash
npm install
```

**Set Up Environment Variables**

Create a new file named `.env` in the root of your project and add the following content:

```env
# Backend
VITE_BACKEND_BASE_URL="http://localhost:8000/api/"

# Cloudinary
VITE_CLOUDINARY_CLOUD_NAME=
VITE_CLOUDINARY_UPLOAD_PRESET=
VITE_CLOUDINARY_UPLOAD_URL=
```

Replace the placeholder values with your real credentials. You can get these by signing up at: [**Cloudinary**](https://jsm.dev/pern-cloudinary).

**Running the Project**

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser to view the project.
